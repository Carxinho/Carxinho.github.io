HTTP请求与响应的组成以及如何使用curl命令

HTTP 请求

HTTP 请求的组成部分

1 动词 路径 协议/版本
2 Key1: value1
2 Key2: value2
2 Key3: value3
2 Content-Type: application/x-www-form-urlencoded
2 Host: www.baidu.com
2 User-Agent: curl/7.54.0
3
4 要上传的数据

关于以上各部分内容的碎碎念：
1.请求最多包含四部分，最少包含三部分。（也就是说第四部分可以为空）
2.第三部分永远都是一个回车（\n），用于分开第二部分和第四部分，必不可少
3.常用动词有 GET POST PUT PATCH DELETE等，一般只获取数据不上传就用GET，当要上传数据时就用POST
4.这里的路径包括「查询参数」，但不包括「锚点」，如果你没有写路径，那么路径默认为 /
5.第 2 部分中的 Content-Type 标注了第 4 部分的格式，两者是一致的

用 Chrome 发请求

1.打开 Network
2.地址栏输入网址
3.在 Network 点击，查看 request，点击「view source」，一定要点否则看不到，这里只会显示请求的前三部分。
4.如果有请求的第四部分，那么在 FormData 或 Payload 里面可以看到

HTTP 响应

HTTP 响应的组成部分

1 协议/版本号 状态码 状态解释
2 Key1: value1
2 Key2: value2
2 Content-Length: 17931
2 Content-Type: text/html
3
4 要下载的内容

关于以上各部分内容的碎碎念：
1.状态码要背，详情请点击这
2.第 2 部分中的 Content-Type 遵循 MIME 规范，而且还标注了第 4 部分的格式，两者是一致的。

用 Chrome 查看响应

1.打开 Network
2.输入网址
3.选中第一个响应
4.查看 Response Headers，点击「view source」，一定要点否则看不到，点击后你能看到响应的前两部分
5.查看 Response 或者 Preview，你会看到响应的第 4 部分

来自小白的git入门blog

本文将通过截图+简单描述来写一下git init、git add和git commit -v这三个命令的使用，有兴趣就继续看下去吧。

git init —— 初始化仓库

要用Git进行版本管理，必须先要初始化仓库。Git使用git init命令初始化仓库。首先建立一个空的目录，初始化成功，该目录下就会生成一个.git目录。.git目录中存放着管理当前目录所需的仓库数据。

git add —— 向暂存区添加文件

要想让文件成为Git的管理对象，需要通过git add 命令将其加入暂存区。暂存区是提交前的一个临时区域。可以看到css文件、js文件与html文件前的标识不一样，这是因为我们通过git add可以把html文件放置在暂存区，使用此命令便可把红色的??变成了绿色的A。

git commit -v —— 查看未提交的文件

git commit -m ‘message’ 可以提交短的消息。如果要提交详细的描述，直接运行git commit 然后编辑器自动运行。如果想要查看是否有文件没有提交，可以运行git commit -v 来查看未提交的文件。

Linux命令之ls、cat、mv、touch的使用方法，以及遇到不明白的命令怎么办？

ls的命令概述

ls命令的英文全名是List即列表的意思，用于显示文件目录列表，和Windows系统下DOS命令dir类似。当执行ls命令时，可列出目录下的非隐藏文件（在Linux下，文件名以点.开头表示该文件为隐藏文件，如.bashrc）。ls命令使用频率非常高，参数也非常多，下面介绍了一部分常用的参数，其他参数可以通过man命令查看帮助手册。

ls的命令语法

–all的缩写，显示所有的文件，包括隐藏文件(以.开头的文件)。

—-directory的缩写，仅列出目录本身，而不是列出目录里的内容列表。

–列出长数据串，显示出文件的属性与权限等数据信息。

cat的命令概述

cat命令用于连接文件或标准输入并打印。这个命令常用来显示文件内容，或者将几个文件连接起来显示。

cat的命令语法

–查看文件内容。

–查看文件中的内容并显示包含查找内容的内容。

mv的命令概述

mv命令是move的缩写，可以用来移动文件或者将文件改名，这也是个常用命令，经常用来备份文件或者目录。

mv的命令语法

–将文件1覆盖到文件2。

–将文件移动到文件夹。

–将文件1覆盖到文件2时会询问是否覆盖。

–将文件1覆盖到文件2时不管文件2是否存在均会强行覆盖。

touch的命令概述

touch指令用来修改文件时间戳，或者新建一个不存在的文件。

touch的命令语法

–当touch一个不存在的文件时会创建此文件，而当touch一个已存在的文件时即修改了文件的修改时间。
